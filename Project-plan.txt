# Stoneman Photography Website - Project Plan

## Project Overview
A clean, simple multi-page portfolio website for Stoneman Photography, an emerging wildlife photographer. The site will serve as a living portfolio that grows with the photographer's career, showcasing wildlife photography work and providing a platform for writing.

---

## Target Audience
- Nature enthusiasts
- Potential clients or collaborators
- Competition judges or galleries
- Personal contacts interested in the photographer's work

---

## Core Goals
1. Showcase wildlife photography in a professional, clean layout
2. Provide a platform for blog writing about photography experiences
3. Make it easy for people to contact the photographer
4. Create a foundation that can grow and evolve over time
5. Keep it simple and maintainable for a coding novice

---

## Technical Requirements

### Technology Stack
- **Pure HTML/CSS/JavaScript** (no frameworks, no build tools)
- Reason: Easy to maintain, no dependencies, beginner-friendly
- Single CSS file for consistent styling across all pages
- Minimal JavaScript for interactive features

### Browser Support
- Modern browsers (Chrome, Firefox, Safari, Edge)
- Full mobile responsiveness (mobile-first approach)

### Hosting
- Compatible with static hosting (GitHub Pages, Netlify, Vercel)
- No server-side processing required
- Can be deployed by simply uploading files

---

## Site Structure

### Pages to Create

#### 1. Home Page (`index.html`)
**Purpose:** First impression, showcase best work, guide visitors to other sections

**Content:**
- Hero section with striking wildlife photo background
- Site title: "Stoneman Photography"
- Tagline: "Capturing the beauty of wildlife and nature"
- Call-to-action button: "View Portfolio"
- Featured work section (3 images)
- Simple, clean design that lets imagery speak

**Design Notes:**
- Full-screen hero image with dark overlay for text readability
- Minimal text - focus on visual impact
- Clear navigation to portfolio

---

#### 2. Portfolio Page (`portfolio.html`)
**Purpose:** Display photography work in organized, browsable format

**Content:**
- Page title: "Portfolio"
- Introduction text: "A collection of wildlife and nature photography, from intimate bird portraits to sweeping landscapes."
- Category filter buttons:
  - "All" (default selected)
  - "Wildlife"
  - "Other"
- Image gallery grid (responsive)
- Lightbox/modal for full-screen image viewing

**Functionality:**
- Category filtering with JavaScript
- Click image to open full-screen lightbox
- Lightbox features:
  - Close button (X)
  - Click outside image to close
  - ESC key to close
  - Image fills screen while maintaining aspect ratio

**Gallery Specs:**
- Start with ~15 image placeholders
- 3-column grid on desktop
- 2-column on tablet
- 1-column on mobile
- Images maintain 4:3 aspect ratio
- Hover effect: slight zoom on images

**Image Distribution:**
- Wildlife category: ~10-12 images (birds, insects, small mammals)
- Other category: ~3-5 images (landscapes, abstract nature)

---

#### 3. Blog Page (`blog.html`)
**Purpose:** List of blog posts about photography experiences

**Content:**
- Page title: "Blog"
- Introduction: "Stories from the field, photography insights, and reflections on the natural world."
- Blog post list (most recent first)

**Each Blog Post Preview Includes:**
- Post title (larger, bold)
- Date published
- Excerpt/preview text (2-3 sentences)
- "Read more →" link

**Sample Posts to Include:**
1. "Finding Beauty in Small Subjects" - About macro photography
2. "Patience in Wildlife Photography" - About waiting for the perfect shot
3. "Preparing for My First Safari" - About upcoming Africa trip

**Design Notes:**
- Clean, readable typography
- Plenty of white space
- Posts separated by subtle lines
- No images in preview (text-focused)

**Future Consideration:**
- Individual blog post pages can be added later
- For now, "Read more" links can go to placeholder pages or be disabled

---

#### 4. About Page (`about.html`)
**Purpose:** Tell the photographer's story, build connection with visitors

**Content:**
- Page title: "About"
- 3-4 paragraphs about:
  - Who the photographer is
  - How they got into wildlife photography (just over a year)
  - Their focus (birds, insects, small wildlife)
  - Their approach and philosophy
  - What's ahead (Africa trip, continued learning)
  
**Suggested Text Structure:**
- Introduction: Brief overview of photographer
- Journey: How they started, what drew them to wildlife photography
- Focus: Main subjects (birds, insects, macro work)
- Philosophy: Approach to capturing images, what they hope to convey
- Future: Growth mindset, upcoming projects

**Design Notes:**
- Centered text column (max-width ~800px)
- Larger, readable font size
- Optional: Single portrait photo of photographer (placeholder for now)

---

#### 5. Contact Page (`contact.html`)
**Purpose:** Provide easy way for people to get in touch

**Content:**
- Page title: "Get in Touch"
- Introduction: "Interested in my work or have a project in mind? I'd love to hear from you."
- Contact form with fields:
  - Name (required)
  - Email (required)
  - Message (required, textarea)
  - Submit button

**Form Behavior:**
- For initial version: Form submission shows alert/message
- Note in code: "Form backend needs to be added (Formspree, Netlify Forms, etc.)"
- Form validates required fields
- Clean, centered layout

**Optional Additional Info:**
- Email address displayed
- Social media links (if applicable)

## Design Specifications

### Color Palette
**Primary:** Black, white, and shades of gray
- Background: White (#ffffff)
- Text: Dark gray (#333333)
- Navigation background: White (#ffffff)
- Footer background: Dark gray (#333333)
- Footer text: White (#ffffff)
- Borders/dividers: Light gray (#e0e0e0)
- Hover states: Medium gray (#666666)

**Rationale:** Minimal, timeless, lets photography be the focus

**Future:** Can add earthy accent colors later (forest green, warm brown)

---

### Typography
**Fonts:**
- System font stack for speed and compatibility:
  ```css
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, sans-serif;
  ```

**Hierarchy:**
- Main heading (h1): 3rem (48px) on desktop, 2rem (32px) on mobile
- Section headings (h2): 2rem (32px) on desktop, 1.5rem (24px) on mobile
- Body text: 1rem (16px), line-height: 1.6
- Navigation: 1rem (16px)

**Font Weights:**
- Headings: 300-400 (light to regular)
- Body: 400 (regular)
- Buttons/CTA: 500-600 (medium to semi-bold)

---

### Layout & Spacing
**Max Content Width:** 1200px (centered)
**Section Padding:** 4rem vertical, 2rem horizontal
**Grid Gaps:** 1.5-2rem
**Mobile Padding:** Reduce to 2rem vertical, 1rem horizontal

---

### Navigation
**Desktop:**
- Fixed top navigation bar
- Logo on left: "Stoneman Photography"
- Menu items on right: Home | Portfolio | Blog | About | Contact
- White background with subtle shadow
- Sticky/fixed to top of page

**Mobile:**
- Hamburger menu icon (☰)
- Menu slides down or appears as overlay when clicked
- Vertical list of links
- Tap outside to close

---

### Responsive Breakpoints
- Desktop: 1200px+
- Tablet: 768px - 1199px
- Mobile: Below 768px

**Key Responsive Behaviors:**
- Gallery: 3 columns → 2 columns → 1 column
- Navigation: Horizontal → Hamburger menu
- Hero text: Larger → Smaller
- Padding: More generous → More compact

## Accessibility Considerations

### Required Features:
1. **Semantic HTML:** Use proper heading hierarchy (h1, h2, h3)
2. **Alt Text:** All images must have descriptive alt attributes
3. **Keyboard Navigation:** 
   - All interactive elements accessible via Tab key
   - Lightbox closeable with ESC key
4. **ARIA Labels:** Add where appropriate (mobile menu, lightbox)
5. **Focus States:** Visible focus indicators on interactive elements
6. **Color Contrast:** Ensure sufficient contrast ratios (WCAG AA)
7. **Form Labels:** All form inputs properly labeled

## Maintenance & Updates

### Adding New Portfolio Images:
1. Save images to `images/portfolio/wildlife/` or `images/portfolio/other/`
2. Open `portfolio.html`
3. Copy existing gallery-item div
4. Update image src and alt text
5. Set correct data-category attribute

### Adding New Blog Posts:
1. Open `blog.html`
2. Copy existing blog-post div
3. Update title, date, and excerpt
4. Add at top of blog-posts container (newest first)

### Updating About Page:
1. Open `about.html`
2. Edit text in `.about-content` section
3. Save and upload
